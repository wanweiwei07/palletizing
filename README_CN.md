# 码垛规划器

面向机器人码垛的三维装箱求解器。给定一组不同尺寸的箱子，计算放置位置和朝向，最大化托盘空间利用率。

## 求解器

### GA3D — GPU 加速遗传算法
- 染色体 = 箱子放置顺序排列 + 旋转标志
- 解码器 = 高度图 + 位压缩体素，贪心 best-fit 放置 + 支撑约束
- 种群在 GPU 上并行解码（融合 Triton 扫描核，每个染色体一个 block）
- 结构化初始化：种群 2/3 按长度从大到小排序（其中一半旋转前 `[0, n/2)` 步，另一半旋转后 `[n/2, n)` 步），剩余 1/3 随机排列，旋转按奇偶交替

### CP-SAT — 约束规划 (fill3d)
- 使用 Google OR-Tools CP-SAT 精确求解
- 基于探针的支撑面积约束
- 适合小规模实例

### 分层规划器 (palletizing)
- 按高度将箱子分组为层
- 每层使用 CP-SAT 做二维排列
- 层间支撑验证

## 快速开始

### GA3D — 多箱型三维装箱 + 可视化
```bash
python visualize_ga3d.py 7      # task_70_seed7
```

### 分层规划器 — 单箱型码垛
```bash
python main.py \
  --pallet-length 1200 --pallet-width 1000 --pallet-max-height 1500 \
  --box-length 300 --box-width 200 --box-height 250 --count 70
```

### 生成任务
```bash
python generate_task.py --count 70 --seed 7
```

## 性能测试

### task_70（70 个混合箱型, pop=1024, 30s）

| Seed | 装载    | 体积利用率 |
|------|---------|-----------|
| 7    | 65/70   | 90.6%     |
| 42   | 67/70   | 87.0%     |
| 99   | 67/70   | 80.9%     |
| 123  | 70/70   | 85.3%     |

### task_lx（35 箱，托盘 1000×1200×1600，RTX 5090，seed=42）

| pop  | 30 秒        | 60 秒        | 每代耗时 |
|------|--------------|--------------|---------|
| 512  | 78.7% (88 代)  | 78.7% (178 代) | 0.34 s  |
| 1024 | 78.1% (60 代)  | 78.1% (120 代) | 0.50 s  |
| 2048 | 78.1% (35 代)  | 78.1% (69 代)  | 0.87 s  |
| 4096 | 78.1% (18 代)  | 78.7% (36 代)  | 1.69 s  |

所有配置均装完全部 35 箱，体积利用率集中在 78.1–78.7%。结构化初始化下，pop=512/30s 已经达到该任务的上限。

体积利用率 = 箱子总体积 /（托盘底面积 × 最高箱子顶部高度）

## 参数调整指南

`GA3DConfig` 参数（见 [ga3d/solver.py](ga3d/solver.py)）：

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `pop_size`           | 2048  | 多样性 vs 单代成本。越大单代越慢，但探索更充分。单代耗时与 pop 近似线性。 |
| `time_limit_seconds` | 60.0  | 总时间预算。GA 运行至已用时间 ≥ 此值。 |
| `mutation_rate`      | 0.15  | 单染色体 swap 变异概率。调高增加探索。 |
| `crossover_rate`     | 0.8   | OX1 交叉概率。 |
| `elite_ratio`        | 0.1   | 每代直接保留的精英比例。 |
| `tournament_k`       | 4     | 锦标赛选择规模。 |
| `min_support_ratio`  | 0.8   | 支撑面积下限，低于此值拒绝放置。提高到 0.9 更安全，降低则放置更紧凑。 |
| `grid_res_xy/z`      | 10 mm | 体素分辨率。越小越精确但越慢（内存/算力呈平方/立方增长）。 |
| `use_triton`         | True  | 融合 Triton 扫描核。仅调试时关闭。 |
| `seed`               | None  | 设置后可复现。 |

**实用建议：**
- 小型 / 规整任务（≤ 50 箱）：`pop=512, time=30s` 通常足够。
- 中型任务（50–100 箱）：`pop=1024–2048, time=30–60s`。
- 困难 / 高混合任务：`pop=2048–4096, time=60–120s`。
- 如果瓶颈在单代耗时，降低 `pop_size`；如果过早收敛，提高 `pop_size` 或 `mutation_rate`。
- task_lx 的测试结果非常平坦，说明瓶颈在解码器/评分函数而非 GA 本身。要进一步提升需改解码器（空隙填充、更精细的支撑评分），调 GA 已无效。
